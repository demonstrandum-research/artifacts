# Borsuk Lean project — toolchain setup

Lean 4 + mathlib project for the formal disproof of Conjecture 3 of
arXiv:2508.20009 (Brose, De Loera, Lopez-Campos, Torres). See
`../../gate0-results.json`, vetted entry "Disproof of the lattice Borsuk
cube-characterization", for the mathematical content and proof plan.

## Versions (pinned)

| Component | Version |
|---|---|
| OS | Windows 11 (native, no WSL) |
| elan | 4.2.3 (installed via `winget install Lean.Elan`; already present at `%USERPROFILE%\.elan\bin`, on user PATH) |
| Lean toolchain | `leanprover/lean4:v4.30.0` (x86_64-w64-windows-gnu), pinned in `lean-toolchain` |
| Lake | 5.0.0-src+d024af0 |
| mathlib | release tag `v4.30.0` (latest stable as of 2026-06-11; master was on v4.31.0-rc2), pinned in `lakefile.toml` via Reservoir scope `leanprover-community` |

The mathlib **release tag** (not master) was chosen so that the build cache is
guaranteed available and the project is reproducible.

## How this was set up

```powershell
# 1. elan was already installed (winget package Lean.Elan, elan 4.2.3).
#    Fresh machine equivalent:  winget install --id Lean.Elan -e
#    Verify:                    elan --version

# 2. Install the pinned toolchain
elan toolchain install leanprover/lean4:v4.30.0

# 3. Project files written by hand (equivalent to `lake new borsuk math`
#    with the mathlib require pinned to rev = "v4.30.0"):
#      lean-toolchain, lakefile.toml, Borsuk.lean, Borsuk/Smoke.lean

# 4. From this directory (problems\p3-moonshot\borsuk\lean):
lake update          # resolves mathlib v4.30.0 + transitive deps, writes lake-manifest.json
                     # NOTE: mathlib's post-update hook ALREADY runs `cache get` —
                     # do NOT start a second `lake exe cache get` while `lake update`
                     # is still running (concurrent downloads race on
                     # %USERPROFILE%\.cache\mathlib and fail with rename error 32).
lake exe cache get   # idempotent; resumes/completes the cache (~1.6 GB of .oleans,
                     # 8459 cache files for mathlib v4.30.0)
lake build           # builds the Borsuk library; with a complete cache only the
                     # local files compile ("Build completed successfully (8477 jobs)")
```

## Day-to-day build commands

```powershell
# elan's shims are on the user PATH; in a fresh shell that predates the
# install, prepend:  $env:Path = "$env:USERPROFILE\.elan\bin;" + $env:Path
cd C:\Users\jacks\source\repos\maths\problems\p3-moonshot\borsuk\lean
lake build                     # incremental build of the Borsuk library
lake env lean Borsuk\Smoke.lean  # elaborate a single file
```

Never run `lake update` casually — it can re-resolve dependencies. The pins in
`lean-toolchain` + `lakefile.toml` + `lake-manifest.json` are the source of
truth. If `.lake` is ever deleted, re-run `lake exe cache get` before
`lake build` to avoid a multi-hour from-source mathlib build.

## Smoke test

`Borsuk/Smoke.lean` (built as part of `lake build`) validates end-to-end:

* `decide` over `Int.gcd` on the pairwise differences of witness A
  `S_A = {(0,0),(1,0),(0,1),(3,5)}` (all primitive),
* `Finset.Icc`/`Int.card_Icc` + `omega` for the `|[0,m] ∩ ℤ| = m+1` count,
* the `convexHull`/`subset_convexHull` API over `ℝ`,
* `interval_cases`/`Nat.mul_le_mul`/`decide` that 7 is not a perfect square
  (note: `decide` alone cannot discharge an unbounded `∀ n : ℕ`).

Axiom audit (`lake env lean scripts\CheckAxioms.lean`): all smoke theorems
depend only on the standard axioms (`propext`, `Classical.choice`,
`Quot.sound`) — no `sorry`, no extra axioms.

## Project layout

```
lean/
  lean-toolchain      # leanprover/lean4:v4.30.0
  lakefile.toml       # package config; mathlib pinned to v4.30.0
  lake-manifest.json  # exact dependency revisions (generated by lake update)
  Borsuk.lean         # root module, imports everything under Borsuk/
  Borsuk/
    Smoke.lean        # end-to-end smoke test (real mathlib content)
  scripts/
    CheckAxioms.lean  # #print axioms audit (run via `lake env lean scripts\CheckAxioms.lean`)
  .lake/              # build artifacts + downloaded deps (gitignored)
```

## Adding new modules

Create `Borsuk\<Name>.lean`, add `import Borsuk.<Name>` to `Borsuk.lean`,
run `lake build`. Keep `import Mathlib` (whole-library import) during
development; it costs ~10 s of import time per file but removes all
import-hunting friction. Use `#print axioms <thm>` in scripts/CheckAxioms.lean
to certify sorry-freeness of the final theorems.
