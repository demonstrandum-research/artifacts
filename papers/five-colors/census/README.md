# Strong-majority edge-colouring — exhaustive small-graph census

Supporting dataset for the *five-colors* flagship (Maj′(G) ≤ 5 for every
admissible graph, kernel-checked in Lean; see `../artifact/`). This directory
contains the **exploratory, SAT-based, exact, dual-checked** finite evidence
referenced in the paper (`../main.md`, "Finite evidence for four colors"). It is
**separate from and does not enter the Lean verification** of the ≤ 5 theorem.

## What was computed

For a graph `G`, `Maj′(G)` is the least palette size admitting a strong-majority
edge-colouring under the frozen `AristotleMaj5.lean` definitions (restated in
full in `SUMMARY.md`). "Admissible" = no adjacent degree pair sums to 3 (i.e. no
adjacent `{1,2}`).

1. **Exhaustive counterexample search through n ≤ 9.** Every connected unlabeled
   graph on ≤ 9 vertices (nauty `geng -c`, 2.8.9): **273,193** isomorphism types,
   of which **268,478** are admissible. Every admissible graph is **4-SAT and
   5-SAT** — zero UNSAT at either palette. Because admissibility and every row
   constraint are component-local and colourings reuse the palette, this proves:
   *every graph on ≤ 9 vertices has a 4-colour strong-majority edge-colouring
   whenever it is admissible.* (`data/exhaustive_n1_7.json`, `_n8.json`, `_n9.json`.)

2. **Exact minimum-palette census through n ≤ 8** (11,640 admissible graphs;
   per-graph records in `data/connected_n{1..8}_maj.jsonl`, distribution in
   `data/census.json` / `data/census_distribution.json`). The observed maximum is
   **Maj′ = 4**; **Maj′ = 5 never occurs**. The unique smallest Maj′ = 4 graph is
   the paw (`graph6 CV`). The `n = 9` exact-value census is banked **honestly as
   partial** (130,039 of 256,838 canonical residue records; also max 4, no 5) and
   is explicitly labelled partial — the *counterexample* search above is
   nonetheless exhaustive through n = 9.

3. **Exhaustive 4-regular census** at orders 10, 12, 14: 59 + 1,544 + 88,168 =
   **89,771** connected unlabeled 4-regular graphs, all 4-SAT and 5-SAT
   (`data/exhaustive_4regular_n10_14.json`).

4. **592 targeted admissible instances**, 2–140 vertices, 1–1,400 edges
   (complete bipartites, regular randoms, cycle/clique blowups, subdivided
   cores), all 4-SAT and 5-SAT. (Regenerable via `search.py targeted --seed
   20260711`; the full 5.4 MB output is not banked here.)

## The calibrated claim

This is **finite evidence for the KKPW four-colour conjecture, not a proof.**
No admissible graph in any tested family required more than four colours; we draw
no conclusion beyond the observation that **no counterexample to the four-colour
conjecture exists among admissible graphs on ≤ 9 vertices, among 4-regular graphs
on ≤ 14 vertices, or in the additional tested families.** In particular any
four-colour failure, if one exists, has ≥ 10 vertices and is not 4-regular
through 14 vertices. The paper's proved theorem is the **five**-colour bound.

## Decision method (exact, no timeouts)

Sequential-counter CNF (exactly-one per edge; at-most `⌊(d(u)+d(v)-2)/2⌋` per row
per colour), solved with CaDiCaL 1.9.5 via python-sat; every returned model is
re-checked directly against every frozen Lean row inequality. Exhaustive runs
used a complete symmetry-aware DFS; UNSAT claims are cross-checked with Glucose
4.2 (exercised by the 3-colour paw calibration instance, which is genuinely
UNSAT — a positive control that the search detects a real palette boundary). The
code never accepts `UNKNOWN`. Full fidelity notes and the frozen definitions are
in `SUMMARY.md`.

## Reproduce / verify

Python 3.11 + `scripts/requirements.txt`, plus nauty `geng` on PATH. From a
working dir with the connected/4-regular `.g6` inputs regenerated by `geng -c`
(and `geng -c -d4 -D4` for the regular census):

```
python scripts/search.py exhaustive connected_n1.g6 ... connected_n9.g6 --output exhaustive_n9.json
python scripts/search.py exhaustive regular4_n10.g6 regular4_n12.g6 regular4_n14.g6 --output exhaustive_4regular_n10_14.json
python scripts/search.py targeted --seed 20260711 --output targeted.json
python scripts/census.py                 # exact minimum-palette census tables
python scripts/verify.py <result.json>   # re-check every stored model directly
```

`search.py certificate` emits DIMACS for any claimed UNSAT graph;
`analyze_instances.py` does distinct-colouring counts. The frozen Lean source is
not modified by any script.

## Files

- `SUMMARY.md` — full verdict, frozen definitions, method, per-order tables,
  extremal witnesses, structural observations.
- `data/census.json` — authoritative exact `n ≤ 8` table + labelled partial `n = 9` bank.
- `data/census_extremal.json` — minimum-order witness (edges, degrees, colouring,
  two-solver boundary) for every value that occurs.
- `data/connected_n{1..8}_maj.jsonl` — per-graph `graph6` → `Maj′` records (exact, n ≤ 8).
- `data/exhaustive_n1_7.json`, `_n8.json`, `_n9.json`,
  `exhaustive_4regular_n10_14.json` — counterexample-search totals + hardest cases.
- `scripts/` — search / census / verify / analysis code + `requirements.txt`.
